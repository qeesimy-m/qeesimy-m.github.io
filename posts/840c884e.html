Github Pages：创建多个页面搭载网页

发表于2023-08-30|更新于2024-01-30
|字数总计:924|阅读时长:3分钟|阅读量:

前言

起因是弄了个Webstack主题的hexo博客，本来可以直接放在原博客上，但每次更新都要重新改文件位置很麻烦，也老是引用不对位置，所以干脆单独弄个仓库上传吧！
阅读本文前，请确保你已经创建过自己的GitHub Pages

本文参考：如何拥有多个GitHub Pages

GitHub Pages 站点的类型

截取GitHub Pages 站点的类型官方原文。

有三种类型的 GitHub Pages 站点：项目、用户和组织。
项目站点连接到 GitHub 上托管的特定项目，用户和组织站点连接到 GitHub.com 上的特定帐户。

发布用户站点，必须创建名为<username>.github.io的个人帐户拥有的存储库。
发布组织站点，必须创建名为<organization>.github.io的组织帐户拥有的存储库。
发布项目站点，储存库的命名无限制。

除非使用的是自定义域，否则用户和组织站点在http(s)://<username>.github.io或http(s)://<organization>.github.io中可用，项目站点在http(s)://<username>.github.io/<repository>或http(s)://<organization>.github.io/<repository>中可用。

只能为GitHub上的每个帐户创建一个用户或组织站点，项目站点没有限制。

创建项目

为方便，以下用[Siteroot]指代用于存放网站项目的文件夹

  1. 新建一个仓库，名称随意。

  2. 设置[Siteroot]/_config.yml下的deploy部分，然后hexo d上传项目到仓库。

  3. 进入仓库主页，点击右面的Settings，找到GitHub Pages部分配置：

  • 选择Branch路径保存
  • 建议勾选 Enforce HTTPS，否则访问时会出现安全警告。
  • 没有出错的话，一个项目主页就建立完成了，可以通过<username>.github.io/<projectname>访问

  1. 配置_config.yml
    完成上述操作后网页可能会不显示css或错误，此时只需要配置[Siteroot]/_config.yml下的URL部分

绑定子域名

这里我新建的项目为qeesimy-m.github.io/nav。若希望通过nav.zeroqsm.top这样更加美观的地址访问，就需要设置和绑定子域名。

  1. 打开域名管理面板，点进域名解析列表（DNS解析管理）
  2. 点击添加记录，需要添加的记录为
记录类型 主机记录 记录值
CNAME 随意 博客地址

  1. 进入仓库主页，点击右面的Settings，点击GitHub Pages,配置Custom domain自定义域名
    填写随意.自定义域名，如我的主机记录填了nav，那么此处就填nav.zeroqsm.top

  2. 添加 CNAME 文件
    这时你的博客资源目录下应该会出现一个CNAME文件，如果没有，就主动添加
    注意没有任何后缀名，而且是cname的大写
    添加文件[Siteroot]/source/CNAME
    文件里面填写刚刚的自定义域名

接下来就可以使用 https://nav.zeroqsm.top 访问我的Webstack主题hexo博客网页了
